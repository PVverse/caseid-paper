📋 Overview

This document reports the full analysis pipeline for the Case-ID annotation study, which evaluated the feasibility of systematically classifying individual case safety reports (ICSRs) for their relevance as evidence of a drug–adverse-event association.

The study examined two adverse-event domains:

  • Impulsivity-related events — reports retrieved for dopamine agonists (DAA), third generation antipsychotics (TGA), methylphenidate, SSRIs, and other drugs.
  • Suicidality-related events — reports retrieved via the MedDRA Standardised MedDRA Query (SMQ) for suicidal and self-injurious behaviour.

Annotation Dimensions

Each report–drug pair was independently annotated by two raters (JF and MF) along two dimensions:

  1. Scope — whether the report is in-scope (drug suspected to be causally involved) or falls into out-of-scope categories.
  2. Relevance — for in-scope cases, whether evidence is suspected, precautionary, or uninformative, and which qualifiers were observed.
✓ Reproducibility

Sections 2 and 3 require the original (non-public) annotation Excel files and are rendered with eval: false for documentation. Sections 4–6 use de-identified shared data and are fully reproducible.

🔍 1. Framework

The framework evaluates retrieved reports across four relevance domains to determine whether they belong to the case series of interest.

Relevance Domains

Relevance Domain What it captures
Exposure Misreported or miscoded drug exposure; falsified or substandard products; placebo-arm reports in blinded trials; drug-name confusion
Event Misreported event due to lack of expertise; miscoded event; mistranslation; non-verified diagnostic criteria
Relationship Bystander (event attributed to different drug); lack of effect or relapse during treatment; reverse causality; discontinuation effects
Setting / Population Miscoded demographic data; non-relevant care setting; use outside routine prescription (off-label, misuse, abuse, poisoning)

Reporter Suspicion Categories

Suspicion Label What it captures
Explicit suspicion The report explicitly or strongly frames the event as an adverse effect
Precautionary The report explicitly frames the event as plausibly not resulting from the drug
Unclear suspicion The report does not contain sufficient information to classify it clearly

Evidentiary Qualifiers

Qualifier Supports Suspicion Weakens Suspicion
Change in pattern/severity New onset, escalation, qualitative change No meaningful change from baseline
Temporality Onset follows treatment initiation or dose change plausibly Timing inconsistent with exposure
Reversibility / dose relation Improves after withdrawal; recurs on rechallenge; dose-related Persists despite withdrawal; no dose relation
Alternative explanations Other explanations explicitly ruled out Another drug or underlying disease better explains event
Explicit causal statement Drug described as causing or likely causing the event Drug described as unrelated

📊 2. Inter-rater Agreement

⚠️ Requires Original Data

This section reads Annotation_JF.xlsx and Annotation_MF.xlsx from the annotation path. These files contain original (potentially identifiable) annotation data and are not distributed. The analysis uses three complementary metrics to quantify rater consistency.

Agreement Metrics

  • Percent agreement — simple proportion of matching labels; interpretable but sensitive to marginal distributions.
  • Cohen's κ (kappa) — chance-corrected agreement; unweighted for scope (nominal), weighted (quadratic) for relevance (ordinal).
  • Krippendorff's α (alpha) — general reliability coefficient handling ordinal data and missing labels.
  • McNemar test — for binary-derived variables, detects systematic directional disagreement between raters.

Disagreement Typology

Beyond global reliability, the analysis characterizes the types of disagreements that occurred:

  • For scope: Recorded which label pair each rater chose
  • For relevance: Distinguished adjacent disagreements (off by 1 level) from far disagreements (off by 2+ levels)
  • Adjudication: Discordant cases exported for consensus review by both raters

📈 3. Annotation Characterisation

⚠️ Requires Original Data

This section reads Annotation_solved.xlsx (adjudicated consensus annotations) and vigiGrade_VB.csv (VigiGrade completeness scores). These are not distributed, but the methodology is documented here.

Solved Annotations

After adjudication, each discordant case receives a consensus label. The analysis computes:

  • Distribution of report types per drug
  • Proportion of in-scope vs. out-of-scope cases
  • Relevance category breakdown (suspected, precautionary, uninformative)

Report Completeness

VigiGrade scores (scale 0–1) are examined to assess whether completeness differs across relevance categories, which would suggest potential bias in annotation patterns.

Feature Patterns

For in-scope reports, the presence/absence of causal evidence elements is analyzed to characterize what distinguishes:

  • Suspected reports (clear supportive evidence)
  • Precautionary reports (evidence present but weakened by contraindications)
  • Uninformative reports (insufficient or equivocal evidence)

💊 5. Case Study 1 — Impulsivity

5.1 Relevance Distribution

The analysis shows the proportion of report types for each drug or drug group:

Drug Groups
  • Positive controls (DAA, TGA) — drugs with established, strong associations with impulse-control disorders
  • Ambiguous (methylphenidate, SSRI) — drugs with mixed or contested evidence
  • Negative — drugs used as comparators with no prior signal for these outcomes

5.2 Sankey Diagram

The Sankey diagram traces each report through a five-stage decision tree:

  1. Retrieved — all reports from the search query
  2. Relevance — Relevant / Irrelevant / Not assessable
  3. Sub-category — for relevant: Suspected / Precautionary / Unclear; for irrelevant: Event / Relationship / Exposure
  4. Role — specific role within sub-category (e.g., Bystander, Miscoding)
  5. Role specification — further subdivision where applicable (e.g., Non-adherence, Withdrawal)

🔗 6. Case Study 2 — Suicidality

6.1 Sankey Diagram

Suicidality reports are traced through the same five-stage decision tree as in Case Study 1. Non-assessable reports appear as a distinct stream at the relevance stage.

6.2 Drug Heatmap

Each column represents a drug; each row represents a report type. Column width reflects total report count per drug (drugs with <3 reports excluded). Color encodes report type frequency.

6.3 Drug × ATC Class Analysis

Distribution of report roles within each ATC first-level class, aggregating over individual drugs. Helps identify whether certain pharmacological classes show systematic patterns in evidence quality.

6.4 Preferred Term Heatmap

Each column represents a MedDRA preferred term (PT); each row represents a report type. Reports with multiple PTs are counted once per PT. The manual PT order reflects clinical sequence from active suicidal behavior through ideation and depressive symptoms.
